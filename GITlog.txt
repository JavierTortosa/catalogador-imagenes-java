    20250822: Feat(filtros): Implementación completa del sistema de filtrado dual
    
    Implementa un sistema de filtrado dual robusto que incluye un Filtro Rápido ("Tornado") temporal y un sistema de Filtros Persistentes multicriterio.
    
    El sistema se basa en un patrón de "checkpoint al vuelo" donde el Filtro Persistente opera sobre una copia inmutable de la lista de archivos actual en el momento de su activación. Se ha refactorizado la lógica de aplicación de filtros para que toda la UI se actualice de forma consistente a través del ListCoordinator, simulando una carga de "carpeta virtual".
    
    Se ha corregido la lógica de la UI para asegurar la coherencia visual entre pestañas y se ha implementado un sistema de agrupamiento funcional para los botones de tipo de filtro.
    
    Principales Cambios:
    
    - feat(filtros): Añadido sistema de Filtro Persistente con lógica AND para múltiples criterios.
    - feat(filtros): Implementado atajo para convertir el Filtro Rápido en un Filtro Persistente.
    - feat(filtros): Añadido soporte para selección de tipo de filtro (Nombre/Carpeta) mediante JToggleButtons.
    - fix(ui): Corregida la lógica de `FilterManager` para aplicar correctamente múltiples filtros positivos.
    - fix(ui): Solucionado el bug que impedía la restauración de la lista maestra al eliminar todos los filtros.
    - fix(ui): Solucionado el problema de renderizado de fondos inconsistentes en las pestañas del panel izquierdo.
    - refactor(controller): Centralizada la lógica de gestión de checkpoints y refresco de UI en `GeneralController`.
    - refactor(builder): Refactorizado `ToolbarBuilder` para manejar `ButtonGroup`s de forma coherente.
